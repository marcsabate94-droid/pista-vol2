ESTRUCTURA DE LA SEGONA PISTA (EXPERIÈNCIA)

1️⃣ El QR del bitllet porta a una pàgina molt neta (tipus mòbil)

Visualment:

  • Fons blanc

  • Tipografia clara

  • Cap imatge pesada

  • Aspecte “document / avís”

Títol gran:

INFORMACIÓ IMPORTANT DEL VOL

Subtítol:

Aquest bitllet no diu on vas…
però ho has de descobrir.

Ja aquí ja sap que el joc continua.

2️⃣ El missatge clau, però amb joc

Text principal (centrat, curt, llegible):

On anem,
hauràs d’afrontar el fred del nord.

Així que hi ha una cosa que
has de posar a la maleta.

I ara ve la gràcia 👇

3️⃣ El “què has de posar a la maleta” NO surt explícit

A sota del text, poses 4 icones o paraules, però només una té sentit:

  • 🕶️ Ulleres de sol

  • 🩱 Banyador

  • 🧥 Abric gruixut

  • 🩴 Xancletes

👉 Ella ha de deduir què toca.

Quan clica (o diu en veu alta) “abric gruixut”, pots:

  • o bé donar-li la següent pista física,

  • o fer que aquesta deducció sigui la clau per continuar (per exemple, obrir una capsa, un calaix, etc.).

2) Publicar-la gratis (tria una d’aquestes dues, la més fàcil)

Opció A — GitHub Pages (free i estable)

  1. Crea un repo nou a GitHub (per exemple: pista-vol)

  2. Puja aquest fitxer com index.html

  3. A Settings → Pages: Source = Deploy from a branch (main / root)

  4. Et donarà una URL del tipus: https://USUARI.github.io/pista-vol/

Opció B — Cloudflare Pages (free i ràpid)

  1. Entra a Cloudflare Pages i crea projecte

  2. Puja el fitxer index.html (o connecta un repo)

  3. Et dona una URL tipus https://nom.pages.dev

Aquí tens el pas a pas per publicar la pista amb GitHub Pages (gratis) i obtenir una URL perquè el QR hi apunti.

0) Abans de començar

Necessites:

  • Un compte de GitHub

  • El fitxer index.html (el que t’he passat)

1) Crear el repositori

  1. Ves a GitHub i inicia sessió.

  2. A dalt a la dreta, clica el botó + → New repository.

  3. Omple:

    • Repository name: per exemple pista-vol

    • Public (important: Pages free és més fàcil així)

    • Marca Add a README file (recomanat)

  4. Clica Create repository.

2) Pujar el fitxer index.html

Opció senzilla (web):

  1. Dins el repo pista-vol, clica Add file → Upload files.

  2. Arrossega el teu index.html (o selecciona’l).

  3. A baix, clica Commit changes.

Important:

  • El fitxer s’ha de dir exactament index.html (minúscules).

  • Ha d’estar a l’arrel del repo (no dins carpetes).

3) Activar GitHub Pages

  1. Al repo, ves a la pestanya Settings.

  2. A la barra lateral esquerra, baixa fins a Pages.

  3. A “Build and deployment”:

    • Source: selecciona Deploy from a branch

    • Branch: selecciona main

    • Folder: selecciona / (root)

  4. Clica Save.

4) Trobar l’URL final

  1. Encara a Settings → Pages, et sortirà una secció “Your site is live at…”

  2. La URL serà tipus:

    • https://EL_TEU_USUARI.github.io/pista-vol/

Si no surt al moment:

  • Recarrega la pàgina (F5)

  • Espera 1–2 minuts (GitHub triga una mica a desplegar)

5) Comprovar que funciona al mòbil

  1. Obre aquesta URL al mòbil.

  2. Hauries de veure la pàgina minimalista amb les opcions.

Si et surt 404:

  • Revisa que el fitxer es digui index.html

  • Que estigui a l’arrel del repo

  • Que Pages estigui a main / root

6) Després: generar el QR

Quan em passis la URL exacta, et dic:

  • la mida ideal del QR per imprimir (perquè escanegi a la primera),

  • i t’indico com col·locar-lo al bitllet.

Si vols fer-ho encara més “pro”: podem posar una URL més curta (per si un dia el QR falla, que pugui escriure-la).
